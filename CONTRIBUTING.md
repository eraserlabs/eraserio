# Contributing to Eraser

We're thrilled you're here! Whether you're fixing a typo, reporting a bug, or building something new — every contribution makes Eraser better. Let's build great diagrams together.

## Ways to Contribute

- **Report bugs** — Found something broken? Open an issue with steps to reproduce
- **Suggest features** — Got an idea? We'd love to hear it — open an issue
- **Submit pull requests** — Code contributions are always welcome

## Getting Started

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/eraser-io.git
   cd eraser-io
   ```
3. Install dependencies:
   ```bash
   cd packages/eraser-mcp
   npm install
   ```

## Pull Request Process

Ready to contribute code? Here's how:

1. Create a feature branch from `main`:
   ```bash
   git checkout -b your-feature-name
   ```
2. Make your changes, keeping them focused and minimal
3. Write clear commit messages
4. Push to your fork and open a pull request
5. Describe what your PR does and link any related issues

We review PRs as quickly as we can — thanks for your patience!

## Code of Conduct

Be kind, be respectful, be constructive. We're all here to build something great together.

## Questions?

Stuck on something? Open an issue or reach out — we're happy to help you get started!
